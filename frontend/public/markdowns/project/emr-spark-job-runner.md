A growing music streaming startup has expanded both its user base and song library, and now plans to setup the data in its data warehouse. Their data is stored in S3, including a directory of JSON logs capturing user activity within the app, and another directory containing JSON metadata about the songs in their catalog.

The goal of this project is to create and deploy an **ETL** job that extracts their data from S3, processes them using **Apache Spark**, and loads the data back into S3 as a set of dimensional tables. This will allow their analytics team to continue finding insights in what songs their users are listening to.

## Tools
AWS Elastic MapReduce (EMR), Apache Spark, Bash, Python


## Steps involved include
- Setup your Spark cluster
- Write the data processing/transformation logic
- Deploy using AWS console
- Deploy using CLI
- Deploy usisng Python AWS SDK (Boto3)

## Dataset
The first dataset is a subset of real data from the [Million Song Dataset](http://millionsongdataset.com/). Each file is in JSON format and contains metadata about a song and the artist of that song. The files are partitioned by the first three letters of each song's track ID.

The second dataset consists of log files in JSON format generated by this [event simulator](https://github.com/Interana/eventsim) based on the songs in the dataset above. These simulate app activity logs from an imaginary music streaming app based on configuration settings.


### Setup your Spark cluster

to setup a Spark cluster.

You can either it up on your local computer or server, you would simply download and install the binaries for Spark, it would require some slightly complex configuration with would need time and if you cluster is too small for the job you would need a bigger machine or more than one to enable the spark program run optimally.

Alternatively, you can use cloud service meaning you would have to pay for virtual server that gets spun up at your request, this servers would come installed with Apache Spark and other programs like Hadoop, HDFS, Zookeper, Livy etc, you can specify the additional programs you need for your data job. This would save time but cost more.

One can deploy using client or cluster mode.

Client mode is typically used for development or prototyping. In this mode, the Spark driver runs on the machine that submits the application (e.g., your laptop or an edge node in the case of cloud EMR). If that machine disconnects or times out, the Spark job can fail because the driver is no longer available. It's more interactive but less reliable for long-running or production jobs.

Cluster mode is used for production deployments. Here, the Spark driver runs on one of the nodes in the cluster. This means the job is independent of the submitting machine, and it continues running even if the client disconnects. The driver manages communication with the executor nodes, schedules tasks, and collects results.

### Deploy using CLI
To setup an EMR cluster and deploy spark jobs using AWS CLI, first you have to make sure you have CLI tool installed.

If you don't have AWS CLI installed, get it [here](https://docs.aws.amazon.com/cli/latest/userguide/getting-started-install.html)

Run the below command to create an EMR cluster with Spark, Hadoop, Hive, Jupyter and  Livy installed. Navigate to the console and you would see the cluster created

```bash
aws emr create-cluster \
    --name "EMR Cluster" \
    --release-label "emr-7.8.0" \
    --use-default-roles \
    --applications Name=Hadoop Name=Hive Name=JupyterEnterpriseGateway Name=Livy Name=Spark \
    --ec2-attributes SubnetId="subnet-eef49ca3" \
    --instance-type m5.xlarge \
    --instance-count 2 \
    --scale-down-behavior "TERMINATE_AT_TASK_COMPLETION" \
    --auto-termination-policy IdleTimeout=3600 \
    --region "us-east-1" \
    --profile manager

```

Run the below command to add your ETL job as a step. You would notice we are referencing the spark script we uploaded to s3

```bash
aws emr add-steps \
    --cluster-id j-32ZWBLZK82C9W \
    --steps Type=CUSTOM_JAR,Name="Spark Program",ActionOnFailure=CONTINUE,Jar="command-runner.jar",Args=["spark-submit","--deploy-mode","client","s3://data-emr-bucket-store/deploy-on-console/spark_job_emr.py"] \
    --region "us-east-1" \
    --profile manager
```

### Deploy using Python AWS SDK


Here we can see how Python `boto3` SDK as used to connect to AWS to create an EMR cluster then the spark job is deployed.

You can also deploy the spark job while creating the cluster or afterwards.

```python

import boto3 
import os 
import configparser

config = configparser.ConfigParser()
config.read('dl.cfg')

os.environ['AWS_ACCESS_KEY_ID']=config['AWS']['AWS_ACCESS_KEY_ID']
os.environ['AWS_SECRET_ACCESS_KEY']=config['AWS']['AWS_SECRET_ACCESS_KEY']


session = boto3.Session(
    aws_access_key_id=os.environ['AWS_ACCESS_KEY_ID'],
    aws_secret_access_key=os.environ['AWS_SECRET_ACCESS_KEY'],
)
client = session.client('emr')

S3_BUCKET = 'data-emr-bucket-store'
S3_KEY = 'deploy-on-console/python/spark_job_emr.py'
S3_URI = 's3://{bucket}/{key}'.format(bucket=S3_BUCKET, key=S3_KEY)

# upload file to an S3 bucket
s3 = session.resource('s3')
s3.meta.client.upload_file("spark_job_emr.py", S3_BUCKET, S3_KEY)

# use the client object to specify the parameter and deploy the cluster and job
response = client.run_job_flow(
    Name="EMR cluster - python deployment",
    ReleaseLabel="emr-7.8.0",
    Instances={
        'InstanceGroups': [
            {
                'Name': 'Master node',
                'Market': 'ON_DEMAND',  # Specify On-Demand for the master node
                'InstanceRole': 'MASTER',
                'InstanceType': 'm5.xlarge',
                'InstanceCount': 1
            },
            {
                'Name': 'Core nodes',
                'Market': 'ON_DEMAND',  # Specify On-Demand for the core nodes
                'InstanceRole': 'CORE',
                'InstanceType': 'm5.xlarge',
                'InstanceCount': 1
            }
        ],
        'KeepJobFlowAliveWhenNoSteps': False,
        'TerminationProtected': False,
    },
    Applications=[
        {
            'Name': 'Spark'
        }
    ],
    Steps=[
        #Here we are adding the job step while creating the cluster, we can also add the step after creating the cluster
    {
        'Name': 'Spark Program',
        'ActionOnFailure': 'CONTINUE',
        'HadoopJarStep': {
            'Jar': 'command-runner.jar',
            'Args': [
                'spark-submit',
                '--deploy-mode', 'client',
                's3://data-emr-bucket-store/deploy-on-console/spark_job_emr.py'
            ]
        }
    }
    ],
    VisibleToAllUsers=True,
    JobFlowRole='EMR_EC2_DefaultRole',
    ServiceRole='EMR_DefaultRole'
)

print(f"Cluster created with the following Job Flow ID: {response['JobFlowId']}")

job_flow_id = response['JobFlowId']
print("Job flow ID:", job_flow_id)


# Once the cluster is deployed you can add step indepedently, it youo need to add other steps
step_response = client.add_job_flow_steps(
    JobFlowId=job_flow_id, 
    Steps=[
    {
        'Name': 'Spark Program',
        'ActionOnFailure': 'CONTINUE',
        'HadoopJarStep': {
            'Jar': 'command-runner.jar',
            'Args': [
                'spark-submit',
                '--deploy-mode', 'cluster',
                's3://data-emr-bucket-store/deploy-on-console/spark_job_emr.py'
            ]
        }
    }
    ]
)

step_ids = step_response['StepIds']

print("Step IDs:", step_ids)


```
